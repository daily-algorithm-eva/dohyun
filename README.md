# DoHyun Algorithm Repository

> **“최소 하루 1문제, 반복 학습은 W1 → W2 → W3”**  
> Issue와 PR을 활용하여 **체계적으로 학습**하는 것을 목표로 합니다.

<br/> 


## 🎯 목표

- 하루 1문제 이상 풀이  
- 오답 회차 관리(W1 ~ W3)를 통한 반복 훈련  
- PR 기반 코드 리뷰로 **코드 품질 향상**  
- Issue 기반으로 **학습 기록 아카이브화**  
- 유형별 솔루션 정리로 **장기 학습 자산 구축**
<br/> 

## 📂 디렉토리 구조 및 브랜치 전략
###  📦 디렉토리 구조
```
solved/
├── boj/
│   ├── wrong-1/      # 1회차 오답
│   ├── wrong-2/      # 2회차 오답
│   ├── wrong-3/      # 3회차 오답
│   └── solved/       # 최종 정답 코드
│       ├── greedy/
│       ├── bfs/
│       ├── dfs/
│       └── ...
└── programmers/
    ├── wrong-1/
    ├── wrong-2/
    ├── wrong-3/
    └── solved/
        ├── greedy/
        ├── bfs/
        ├── dfs/
        └── ...
```

<br/> 

### 🌿 브랜치 전략

| 브랜치명   | 역할            |
|-----------|------------------|
| main      | 최종 정리본      |
| solved    | 맞춘 문제 관리   |
| wrong-1   | 첫 번째 오답     |
| wrong-2   | 두 번째 오답     |
| wrong-3   | 세 번째 오답     |

<br/>

### 문제 단위 작업 브랜치

- BOJ: boj/<문제번호> 예) boj/1123
- Programmers: prog/<문제번호> 예) prog/1103

<br/>


## 🔄 문제 풀이 프로세스 (Flow)

- **문제 시도**
문제 번호 기준으로 작업 브랜치 생성 예) boj/1123  
- 결과에 따른 PR 작성
- **오답일 경우**
  - PR 머지 후 Issue 생성  
  - 예시: [W1] BOJ 1102
  - 일정 기간 후 재도전
  - 재도전에서도 틀리면
    - 2회차 오답 → wrong-2 브랜치로 PR
    - 3회차 오답 → wrong-3 브랜치로 PR
      <br/>
- **정답을 맞춘 경우**
  - solved 브랜치로 PR 제출
  - 파일을 문제 유형별 폴더(예: greedy/, bfs/, dfs/…)로 이동해 정리
